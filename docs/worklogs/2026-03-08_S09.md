## Work Done

### Two-Finger Gripper (r_joint + l_joint)
- Simplified gripper from 6-joint (4-bar linkage) to 2-joint (r_joint + l_joint with mimic multiplier=-1)
- Added bridge topics for `/arm/gripper/r_cmd` and `/arm/gripper/l_cmd`
- 4-bar linkage is fundamentally broken in DART: mimic constraints not supported, and independently controlling all 6 joints causes ODE collision crash

### Simulated Grasp via Teleport
- After gripper closes, set `grasped_object` to track which Gazebo model is held
- During `move_to()`, teleport the object to EE position every 0.5s to keep it attached
- On place: open gripper, set cube to ground level (z=0.025), clear grasped_object
- Standard approach in Gazebo demos where physics-based grasping is impractical

### EE World Position Calculation
- TF2 lookup: `base_footprint` → `end_effector_link` gives EE offset in robot frame
- Robot world pose from `gz model --pose` (odom reports 0,0,0 at startup regardless of spawn)
- Rotate EE offset by robot yaw to get world coordinates

### MultiThreadedExecutor
- Single-threaded executor blocked ALL callbacks during `time.sleep()` in pick/place
- Switched to `MultiThreadedExecutor(num_threads=4)` + `ReentrantCallbackGroup` for command sub
- TF and joint_state callbacks now run during blocking sequences

### Teleport Speed Optimization
- Created `/tmp/gz_fast.sh` wrapper with pre-exported `GZ_CONFIG_PATH` and `LD_LIBRARY_PATH`
- Avoids sourcing full `/opt/ros/jazzy/setup.bash` on every subprocess call
- Cache robot pose at start of pick/place (robot stationary during arm operations)
- Teleport interval: ~0.8s (down from ~5s before optimization)

### DC5 Verification
- **Pick**: Cube lifted from z=0.025 to z=0.472 — delta of 0.447m (SUCCESS)
- **Place**: Cube placed back at z=0.025 at (2.370, 1.999) — correctly on ground
