## Work Done

### Busy Flag Race Condition
The `self.busy = True` was set inside `_process_command` (thread target), but
`_user_command_cb` checked `self.busy` before the thread could set it. Two rapid
commands could both pass the check and spawn parallel threads.

Fix: Set `self.busy = True` in `_user_command_cb` before `thread.start()`.

### Rotation Scan for Detection
The camera has a 90┬░ HFOV and the 5cm cubes require the robot to face them directly.
At room center (2,2), the red cube at (2.5,2.5) is 45┬░ off the initial heading.

Added 360┬░ rotation scan to `_tool_detect`: rotate at 0.5 rad/s, check `/detections`
every 1s. When a detection is found, stop and re-verify after 0.5s settling.

### AMCL Settling After Scan
Navigation after rotation scan failed (status 6) because AMCL localization was
disrupted by the continuous rotation. Added 2s settling delay after scan stops
to let AMCL particle filter converge before subsequent navigation.

### Garbled Tool Call Recovery
qwen2.5:7b sometimes outputs tool calls as text instead of structured function calls:
```
portun {"name": "pick_up", "arguments": {"object_name": "red_cube"}}
```
Added `_parse_text_tool_call()` that uses regex to extract `name` and `arguments`
from garbled text output, validates against known tool names, and executes the
recovered tool call. The result is fed back as a user message (since there's no
`tool_call_id` to reference).
