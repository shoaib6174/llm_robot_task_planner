### Perception Node
- Created HSV color-based detection (not YOLO — COCO-pretrained models don't detect "cubes")
- Detects red, green, blue, yellow cubes via HSV segmentation
- Morphological cleanup (open + close) to reduce noise
- Publishes JSON detections on `/detections` and annotated images on `/detections/image`
- Depth estimation via median of 5×5 patch around detection center

### Arm Stability Fix
- Added JointPositionController plugins for arm joints 1–5 (PID: p=200, i=5, d=20)
- Added `<dynamics damping="10.0" friction="5.0"/>` to all arm revolute joints
- This prevents arm from falling under gravity at startup, which was pushing the robot off position

- Increased cubes from 3cm to 5cm (PRD spec: 5–8cm) for better visibility

## Decisions Made
| Decision | Rationale |
|----------|-----------|
| HSV color detection instead of YOLO | COCO-pretrained YOLO doesn't detect "cubes"; HSV is reliable in simulation with known colors |
| Fixed body camera instead of arm-only | Arm camera requires stable positioning; body cam gives consistent perception for navigation |
| Attach sensor to body_cam_link not optical_frame | Gazebo renders along +x of attached frame; optical frame rotation caused incorrect view |
| Joint damping=10.0, friction=5.0 | Simulates real servo resistance; prevents gravity collapse without JointPositionController commands |
| Cube size 5cm (not 3cm) | PRD spec says 5–8cm; 3cm cubes too small to detect at room distances |
